## ¿Qué es un literature review?

En investigación generativa, un literature review —también llamado desk research— consiste en recopilar, analizar y resumir información publicada previamente en fuentes como artículos, papers, videos y bases de datos. No se genera información nueva: se estudian investigaciones ya realizadas para construir una base de conocimiento sólida antes de salir a campo.

<Pullquote>
  "Un buen literature review es el punto de partida ideal para cualquier proyecto de investigación: ordena lo que ya se sabe y abre camino para descubrir lo que aún no."
</Pullquote>

]} />

## Cuándo usarlo

## Cómo hacerlo paso a paso

### 1. Definí el tema y tus preguntas de investigación

Elegí un foco claro. Las preguntas deben ser lo suficientemente amplias para explorar, pero concretas para mantener la revisión manejable.

<CaseStudy
  statement="Ejemplo: investigación sobre agricultura sostenible"
  items={[
    { label: 'Pregunta exploratoria', text: '¿Qué técnicas se están usando en agricultura sostenible a nivel global?' },
    { label: 'Pregunta específica', text: '¿Qué desafíos concretos enfrentan los agricultores al adoptar nuevas tecnologías?' },
    { label: 'Por qué funciona', text: 'La primera abre el panorama; la segunda enfoca la revisión en lo que el equipo necesita saber para tomar decisiones.' },
  ]}
/>

### 2. Reuní y organizá tus fuentes

No todas las fuentes tienen el mismo peso. Combiná fuentes internas y externas para obtener una visión completa.

<CriteriaList items={[
  { name: 'Fuentes internas', desc: 'Entrevistas anteriores, bases de datos de la empresa, feedback de usuarios, reportes internos.' },
  { name: 'Fuentes académicas', desc: 'Google Scholar, ProQuest, JSTOR. Priorizá publicaciones recientes y con metodología clara.' },
  { name: 'Fuentes especializadas', desc: 'Blogs de referencia como Nielsen Norman Group o UX Booth, libros del área, conferencias.' },
  { name: 'Fuentes sociales', desc: 'Redes sociales, foros, comunidades de usuarios. Útiles para capturar lenguaje y percepción real.' },
]} />

<Callout>
  **Organizá en una hoja de cálculo:** registrá autor, año, fuente, método usado, hallazgos principales y limitaciones. Sin esta estructura, el volumen de información se vuelve inmanejable en el paso de análisis.
</Callout>

### 3. Analizá el contenido

Una vez reunidas las fuentes, el objetivo es encontrar la señal dentro del ruido.

<CriteriaList items={[
  { name: 'Buscá coincidencias y contradicciones', desc: '¿Qué dicen todas las fuentes? ¿Dónde hay desacuerdo? Las contradicciones suelen señalar áreas donde se necesita investigación primaria.' },
  { name: 'Identificá vacíos de información', desc: '¿Sobre qué no hay datos suficientes? Esos vacíos son oportunidades para investigar y diferenciarte.' },
  { name: 'Observá el enfoque metodológico', desc: '¿Las fuentes son cualitativas, cuantitativas o mixtas? Eso condiciona qué tipo de conclusiones podés extraer.' },
]} />

### 4. Escribí tu informe

<CriteriaList items={[
  { name: 'Introducción', desc: 'Propósito de la revisión, preguntas que guiaron la búsqueda y alcance del estudio.' },
  { name: 'Resumen de fuentes y temas', desc: 'Principales ideas, hallazgos comunes y patrones identificados entre las fuentes.' },
  { name: 'Limitaciones', desc: 'Sesgos detectados, carencias de información o áreas poco exploradas que afectan las conclusiones.' },
  { name: 'Recomendaciones', desc: 'Qué investigar en profundidad a continuación y cómo usar los insights para guiar decisiones estratégicas del equipo.' },
]} />
